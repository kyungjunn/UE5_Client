# 작업 내역 — UE5 계정 서버 통신 클라이언트

작성일: 2026-07-15

## 개요

언리얼 엔진 5.7 클라이언트(`UEClient_20260715`)에서 계정 백엔드 서버(FastAPI + MySQL)와
REST(JSON)로 통신하는 **C++ 클라이언트**를 구현했다. 회원가입 · 로그인 · 내 정보 조회 ·
회원탈퇴 4개 기능을 제공한다.

기존 프로젝트는 블루프린트 전용(Source 없음)이었으며, 이번 작업으로 **C++ 프로젝트로 전환**했다.

## 확정된 설계 결정

| 항목 | 결정 |
|------|------|
| 구현 방식 | C++ (언리얼 HTTP 모듈) |
| 아키텍처 | `UGameInstanceSubsystem` 단일 클래스 (A안) |
| 호출 방식 | `BlueprintCallable` 함수 + 콘솔 명령어 (둘 다) |
| 토큰 보관 | JWT 메모리 보관 (재시작 시 소멸) |
| 서버 주소 | 기본 `http://127.0.0.1:8000` (`SetBaseUrl`로 변경) |

## 서버 계약 (서버 소스에서 직접 확인)

| API | 요청 본문 | 성공 응답 | 실패 |
|-----|-----------|-----------|------|
| POST `/auth/register` | `{email, nickname(2~50), password(8~128)}` | 201 `{id, email, nickname, created_at}` | 409 / 422 `{detail}` |
| POST `/auth/login` | `{email, password}` | 200 `{access_token, token_type}` | 401 `{detail}` |
| GET `/auth/me` | (Bearer) | 200 `{id, email, nickname, created_at}` | 401 |
| DELETE `/auth/withdraw` | (Bearer) | 204 (본문 없음) | 401 |

- 인증: `Authorization: Bearer <access_token>` (JWT, HS256)
- 에러 본문: FastAPI 표준 `{"detail": "..."}` (문자열) / 422는 `{"detail": [{"msg": ...}]}` (배열)

## 구현 결과

### 데이터 구조
- `FAccountUser` (USTRUCT, BlueprintType) — `Id`, `Email`, `Nickname`, `CreatedAt`

### 델리게이트 (다이내믹 멀티캐스트, BP 바인딩용)
- `FAccountSimpleResult(bool, int32, FString)` — 로그인 / 탈퇴
- `FAccountUserResult(bool, int32, FString, FAccountUser)` — 회원가입 / 내 정보

### UAccountSubsystem (UGameInstanceSubsystem)
- 상태: `BaseUrl`, `AuthToken`(메모리)
- `UFUNCTION(BlueprintCallable)`: `Register` / `Login` / `GetMe` / `Withdraw` / `SetBaseUrl`
- `BlueprintAssignable` 델리게이트 4개: `OnRegisterCompleted` / `OnLoginCompleted` / `OnMeCompleted` / `OnWithdrawCompleted`
- Me/Withdraw는 토큰 없으면 요청 없이 즉시 실패 처리

### 콘솔 명령어 (`Initialize`에서 `IConsoleManager`로 등록)
- `Account.Register <email> <nickname> <password>`
- `Account.Login <email> <password>`
- `Account.Me`
- `Account.Withdraw`
- `Account.SetBaseUrl <url>`
- 결과는 델리게이트 구독으로 `UE_LOG` 자동 출력

### 에러 처리
- 연결 실패 → `bSuccess=false, StatusCode=0, "서버에 연결할 수 없습니다."`
- 2xx → 성공 / 비2xx → `detail` 파싱 (문자열 그대로 / 422 배열은 첫 `msg`)
- 인증 필요 API인데 토큰 없음 → 요청 없이 `"로그인이 필요합니다."`
- 204(withdraw)는 본문 파싱 없이 성공 처리

## 2차 작업: 로그인 / 회원가입 UserWidget 연동 (C++ 베이스)

`UAccountSubsystem`을 호출하고 결과를 UI에 반영하는 UserWidget C++ 베이스 클래스를 추가했다.

### 구조
- `UAuthUserWidgetBase (UUserWidget, Abstract)` — 공통 베이스
  - `StatusText` (BindWidgetOptional) 상태 메시지 출력
  - `GetAccountSubsystem()`, `SetStatus(Message, bIsError)` (성공 초록 / 실패 빨강)
- `ULoginUserWidget` — BindWidget: `EmailInput`, `PasswordInput`, `LoginButton`, `SignupButton`
  - `RegisterWidgetClass` (TSubclassOf, EditAnywhere) — 띄울 회원가입 위젯 지정
  - BlueprintImplementableEvent: `OnLoginFinished(bSuccess, Message)`
- `URegisterUserWidget` — BindWidget: `EmailInput`, `NicknameInput`, `PasswordInput`, `RegisterButton`, `ExitButton`
  - BlueprintImplementableEvent: `OnRegisterFinished(bSuccess, Message)`

### 동작
- 버튼 클릭 → 입력값 공백 검사 → 버튼 비활성화 + "처리 중..." → 서브시스템 호출
- 완료 델리게이트 수신 → 버튼 재활성화 → `StatusText` 반영 → `On...Finished` BP 이벤트 호출
- `NativeConstruct`에서 델리게이트 구독, `NativeDestruct`에서 해제
- Build.cs 의존성에 `UMG`, `Slate`, `SlateCore` 추가

### 화면 전환 (Signup / Exit 버튼)
- 로그인 화면의 `SignupButton` → `RegisterWidgetClass`로 지정된 위젯을 생성해 `AddToViewport`(오버레이)
- 회원가입 화면의 `ExitButton` → `RemoveFromParent()`로 자신 제거 → 밑에 깔린 Login 화면 복귀
- 즉, Signup 클릭 → WBP_Register 오버레이 → X 클릭 → Login 복귀

### 블루프린트 연동
1. `LoginUserWidget` / `RegisterUserWidget`을 부모로 하는 Widget Blueprint 생성
2. 디자이너에서 위 BindWidget 이름과 **정확히 일치**하는 자식 위젯 배치
   - Login: `SignupButton` 버튼 추가 / Register: `ExitButton` 버튼 추가
3. `WBP_Login`의 Class Defaults → Account 카테고리 → `Register Widget Class`에 `WBP_Register` 지정
4. 성공 후 화면 전환 등은 `On Login Finished` / `On Register Finished` 이벤트에 연결

## 엔드투엔드 테스트 방법

1. 서버 기동 (`C:/work/VibeCoding`, MySQL `game_account` DB 켜진 상태):
   ```
   uvicorn app.main:app --reload
   ```
2. UE 에디터에서 `UEClient_20260715.uproject` 열기 (재빌드 프롬프트 수락)
3. Play(PIE) 후 콘솔(백틱 `` ` `` 키)에서 순서대로 실행:
   ```
   Account.Register test@a.com 홍길동 password123
   Account.Login test@a.com password123
   Account.Me
   Account.Withdraw
   Account.Login test@a.com password123      # 탈퇴 후 → 401 기대
   ```
   결과는 Output Log 창에 `[Account] ...`로 출력.

기대 동작: 가입 201 → 중복 409 → 로그인 → 내 정보 → 탈퇴 204 → 재로그인 401.
