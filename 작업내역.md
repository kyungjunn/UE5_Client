# 작업 내역 — UE5 멀티플레이 클라이언트

작성일: 2026-07-15 / 최종 갱신: 2026-07-23

## 개요

언리얼 엔진 5.7 클라이언트(`UEClient_20260715`). 블루프린트 전용 프로젝트를 **C++ 프로젝트로 전환**하고
아래 시스템을 구현했다.

- **계정 시스템** — FastAPI + MySQL 백엔드(Ubuntu)와 REST(JSON) 통신: 회원가입 · 로그인 · 내 정보 · 회원탈퇴
- **로그인/회원가입 UI** — UserWidget C++ 베이스 + 화면 전환
- **로비/세션(방) 시스템** — OnlineSubsystem(NULL/LAN) 리슨서버: 방 생성/검색/참가, 닉네임 복제
- **NPC 채팅(LLM)** — 로컬 llama.cpp + Gemma 서버와 통신, 성격 프롬프트 조립, 로비 NPC "세라"
- **방 준비/시작 → InGame** — 전원 준비 후 시작, ThirdPerson 레벨에서 필드 NPC(F 키 대화)
- **UI 에셋 파이프라인** — 에디터 전용 C++ 도구 + Python 커맨드릿으로 위젯/BP/맵 자동 생성

## 계정 서버 통신 (`UAccountSubsystem`)

### 서버 계약
인증: `Authorization: Bearer <access_token>` (JWT, HS256)

| API | 요청 본문 | 성공 응답 | 실패 |
|-----|-----------|-----------|------|
| POST `/auth/register` | `{email, nickname(2~50), password(8~128)}` | 201 `{id, email, nickname, created_at}` | 409 / 422 `{detail}` |
| POST `/auth/login` | `{email, password}` | 200 `{access_token, token_type}` | 401 `{detail}` |
| GET `/auth/me` | (Bearer) | 200 `{id, email, nickname, created_at}` | 401 |
| DELETE `/auth/withdraw` | (Bearer) | 204 (본문 없음) | 401 |

### 구현
- `FAccountUser` (USTRUCT) / 다이내믹 멀티캐스트 델리게이트 4종 (`OnRegisterCompleted` 등, BP 바인딩용)
- `Register` / `Login` / `GetMe` / `Withdraw` / `SetBaseUrl` (BlueprintCallable)
- 콘솔 명령: `Account.Register` `Account.Login` `Account.Me` `Account.Withdraw` `Account.SetBaseUrl` — 결과는 `[Account]` 로그
- 에러 처리: 연결 실패(StatusCode 0) / 비2xx `detail` 파싱(422 배열은 첫 `msg`) / 토큰 없는 인증 API 는 즉시 실패
- PIE 다중 인스턴스에서 콘솔 명령 중복 등록 → 종료 시 이중 해제 크래시 방지 가드(`FindConsoleObject`)

## 로그인 / 회원가입 위젯

- `UAuthUserWidgetBase`(Abstract): `StatusText`(선택) 상태 출력, 성공 초록/실패 빨강
- `ULoginUserWidget`: `EmailInput` `PasswordInput` `LoginButton` `SignupButton` + `RegisterWidgetClass`
  — 로그인 성공 → `GetMe` → 닉네임 캐시 → `OpenLevel(Lobby)`
- `URegisterUserWidget`: `EmailInput` `NicknameInput` `PasswordInput` `RegisterButton` `ExitButton`

- 동작: 클릭 → 공백 검사 → 버튼 비활성 + "처리 중..." → 완료 델리게이트 → 재활성 + 상태 반영 + `On...Finished` BP 이벤트

## 로비 / 세션(방) 시스템

### 레벨 흐름
```
[Login] --로그인 성공(GetMe)--> [Lobby: 프로필 + 방만들기 + RoomList + NPC 채팅]
  방 만들기 → CreateSession(max 4) → ServerTravel("Room?listen")
  방 참가   → JoinSession → ClientTravel → [Room (listen server)]
[Room] 닉네임 목록 + 준비/시작 → 전원 준비 시 ServerTravel("InGame?listen")
[InGame] ThirdPerson 이동 + 필드 NPC 대화 (F 키)
```

### 클래스
| 클래스 | 역할 |
|--------|------|
| `UUEClientGameInstance` | 로그인 닉네임 캐시 (레벨 이동해도 유지) |
| `UGameSessionSubsystem` | 세션 래핑: `HostRoom`/`FindRooms`/`JoinRoomById`/`LeaveRoom`, `FRoomInfo`, 완료 델리게이트 |
| `ALobbyGameMode` | Room 맵 게임모드 (폰 없음, PC/PS 지정) |
| `ALobbyPlayerController` | 위젯 표시(`LobbyWidgetClass`) + 커서 + `FInputModeUIOnly`, `ServerSetNickname`/`ServerSetReady`/`ServerRequestStartGame` RPC |
| `ALobbyPlayerState` | 복제되는 `Nickname`, `bIsReady` |
| `ULobbyWidget` | 프로필 + 방만들기 + RoomList(동기화) + 우측 NPC 채팅 패널 |
| `URoomListEntryWidget` | 방 한 줄 (꽉 차면 Join 비활성) |
| `URoomWidget` | 닉네임/준비 상태 목록 + 준비(클라)/시작(호스트)/나가기 |

### 주요 동작
- 방 세션: `NumPublicConnections=4`, `bIsLANMatch=true`, 커스텀 설정 `ROOMNAME`/`HOSTNAME`
- RoomList: `RoomId` 기준 동기화 — 새 방 추가·기존 갱신·**최신 검색에 없는 방 제거**
- OSS 는 `Online::GetSubsystem(GetWorld())` 로 **World 기반 조회** — PIE 다중창에서 인스턴스별 세션 분리
  (전역 `IOnlineSubsystem::Get()` 은 광고/검색 충돌 + 유령 세션 원인이었음), 검색 진행 중 재요청 가드
- 준비/시작: 클라 준비 토글 → `bIsReady` 복제, 호스트는 자신 제외 전원 준비 시 시작 가능(혼자면 즉시)
- 방 안 닉네임: `ServerSetNickname` RPC → PlayerState 복제, 위젯이 0.5초 타이머로 `PlayerArray` 갱신

### 설정
- `Build.cs`: `OnlineSubsystem`, `OnlineSubsystemUtils`, `EnhancedInput`, `InputCore`, `HTTP`, `Json` 등
- `.uproject`: `OnlineSubsystemNull`, `OnlineSubsystemUtils`, `PythonScriptPlugin` 활성화
- `DefaultEngine.ini`: `GameInstanceClass`, `[OnlineSubsystem] DefaultPlatformService=Null`

## UI 에셋 파이프라인 (`ULobbyAssetGenerator`)

UE 5.7 Python 에서는 `WidgetBlueprint.WidgetTree`/`RootWidget` 이 protected 라 순수 Python 으로
위젯 트리 구성이 불가 → **에디터 전용 C++ 도구**(`WITH_EDITOR`)를 만들어 Python 커맨드릿에서 호출한다.

- `GenerateLobbyAssets()` — WBP_RoomEntry(Border 루트) / WBP_Room(준비·시작·나가기) / WBP_Lobby(2컬럼: 로비 + 채팅 슬롯)
- `GenerateRoomWiring()` — BP_RoomPC(WBP_Room) / BP_RoomGM
- `GenerateNpcChatAssets()` — WBP_NpcChat + WBP_Lobby CDO 배선
- `GenerateInGameAssets()` — IA/IMC_Interact, WBP_InGameChat, WBP_InteractPrompt, BP_InGamePC, BP_InGameGM
- 기존 에셋은 위젯 트리 **in-place 재구성**(UClass 유지 → 외부 참조 보존), 멱등 실행 가능
- CDO 기본값은 리플렉션(`FClassProperty`/`FObjectProperty`)으로 설정
- 전 화면 위젯은 1920×1080 기준 풀스크린 앵커(0,0→1,1) 레이아웃
- InGame 맵 복제/NPC 배치/GameMode Override 는 Python 드라이버가 수행

## NPC 채팅 — 로컬 LLM (llama.cpp + Gemma)

### 통신 (`UNpcChatSubsystem`)
- `FNpcProfile` 구조체: `Name` / `Personality`(성격 지침) / `StatusLine`(상태) / `PortraitColor`
- **페이로드 조립**: system 메시지 = `"당신은 '{이름}'입니다. {성격}\n현재 상태: {상태}\n답변은 한국어로 2~3문장 이내로 짧게 하세요."` + user 메시지 → `POST /v1/chat/completions` (`max_tokens 1024`, `temperature 0.7`)
- **비동기 + 델리게이트**: 완료 시 `OnChatResponseReceived(bool, NpcName, Message)` 브로드캐스트
- 타임아웃 180초(`SetTimeout` + `SetActivityTimeout`) — 로컬 LLM 은 생성 완료까지 무응답이므로 둘 다 필요
- 진행 중 요청 가드, 콘솔 명령 `Npc.Chat <msg>` / `Npc.SetLlmUrl <url>`
- Gemma thinking 모델: content 가 비면 실패 처리 (서버를 `--reasoning-budget 0` 으로 실행 권장)

### 로비 NPC "세라"
- `UNpcChatWidget`: 초상화(색+이니셜)/이름, 대화 로그(ScrollBox), 입력+전송(Enter), 대기 "..." → 응답 교체
- WBP_Lobby 우측 고정폭 패널(`ChatPanelSlot`)에 삽입, 우상단 = 초상화+이름

## 해결한 주요 이슈

| 이슈 | 원인 → 해결 |
|------|------------|
| 방 만들기 → 로그인 화면 등장 | Room 맵 부재로 `ServerTravel` 실패 → `GameDefaultMap` 폴백. Room 맵 생성 |
| 방 리스트 항목이 안 보임 | 엔트리 루트 CanvasPanel 은 ScrollBox 안에서 desired size 0 → Border 루트로 교체 |
| PIE 2창 종료 크래시 | 콘솔 명령 중복 등록 → 이중 해제. 첫 인스턴스만 등록하는 가드 |
| 다른 클라에서 방 안 보임 / 유령 방 | 전역 OSS 공유 → World 기반 OSS 조회 + 검색 가드 + 리스트 동기화(사라진 방 제거) |
| 서버 500 (회원가입/로그인) | DB `users.deleted_at` 컬럼 누락 → `ALTER TABLE` (서버 측) |
| `localhost` LLM 연결 실패 | UE HTTP 가 IPv6(::1)로 해석, llama.cpp 는 IPv4만 리슨 → `127.0.0.1` 사용 |
| LLM 60초 타임아웃 | 생성 완료까지 무응답 + UE 무활동 타임아웃 → `SetTimeout/SetActivityTimeout 180` |
| BP CDO 배선이 저장 안 됨 | 커맨드릿 `save_dirty_packages` 로는 리플렉션 CDO 수정 누락 → `save_directory(only_if_is_dirty=False)` 강제 저장 + 별도 프로세스 재검증 |
| IMC `MapKey()` 미반영 | C++/Python 모두 저장에 반영 안 됨 → `mappings` 프로퍼티 직접 설정 |
| NPC 프롬프트 미표시 | 폰 스폰 시 초기 오버랩이 빙의 전 발생해 유실 → 저빈도 거리 체크로 교체 |
| 채팅 패널 높이 40px | 비스트레치 Y 앵커에서 `Offsets.Bottom` = 여백이 아니라 높이 → 320 지정 |

## 엔드투엔드 테스트 방법

1. 계정 서버 기동 (Ubuntu `~/VibeCoding`, MySQL `game_account` DB 켜진 상태):
   ```
   uvicorn app.main:app --host 0.0.0.0 --port 8088
   ```
2. LLM 서버 기동 (같은 PC): llama.cpp `llama-server` + Gemma, 포트 8080 (`--reasoning-budget 0` 권장)
3. UE 에디터에서 프로젝트 열기 → Play 설정: Number of Players = 2, Net Mode = **Play As Listen Server**
4. 각 창에서 로그인 → Lobby (우측 세라와 채팅 가능)
5. 호스트 창: 방 이름 입력 → 방 만들기 / 클라 창: 새로고침 → 참가
6. Room: 클라 준비 → 호스트 시작 → 전원 InGame 이동
7. InGame: WASD 이동 → NPC 접근 → "[F] 대화 시작" → F → 입력창 클릭 후 질문 → 응답(수십 초) → ESC 또는 (포커스 해제 후) F 로 닫기 → 이동 재개

콘솔 명령 검증: `` ` `` 키 → `Account.Login ...`, `Npc.Chat 안녕하세요` → Output Log 의 `[Account]` / `[NpcChat]`

참고: LLM 주소가 `127.0.0.1` 이므로 다른 PC 클라이언트는 LLM 접근 불가 (같은 PC 다중 실행은 정상). 필요 시 `Npc.SetLlmUrl` 로 변경.
