작성일: 2026-07-15 / 최종 갱신: 2026-07-23

개요

언리얼 엔진 5.7 클라이언트(UEClient_20260715). 블루프린트 전용 프로젝트를 C++ 프로젝트로 전환하고 아래 시스템을 구현했다.

  • 계정 시스템 — FastAPI + MySQL 백엔드(Ubuntu)와 REST(JSON) 통신: 회원가입 · 로그인 · 내 정보 · 회원탈퇴
  • 로그인/회원가입 UI — UserWidget C++ 베이스 + 화면 전환
  • 로비/세션(방) 시스템 — OnlineSubsystem(NULL/LAN) 리슨서버: 방 생성/검색/참가, 닉네임 복제
  • NPC 채팅(LLM) — 로컬 llama.cpp + Gemma 서버와 통신, 성격 프롬프트 조립, 로비 NPC "세라"
  • 방 준비/시작 → InGame — 전원 준비 후 시작, ThirdPerson 레벨에서 필드 NPC(F 키 대화)
  • UI 에셋 파이프라인 — 에디터 전용 C++ 도구 + Python 커맨드릿으로 위젯/BP/맵 자동 생성

로비 / 세션(방) 시스템

레벨 흐름

[Login] --로그인 성공(GetMe)--> [Lobby: 프로필 + 방만들기 + RoomList + NPC 채팅]
  방 만들기 → CreateSession(max 4) → ServerTravel("Room?listen")
  방 참가   → JoinSession → ClientTravel → [Room (listen server)]
[Room] 닉네임 목록 + 준비/시작 → 전원 준비 시 ServerTravel("InGame?listen")
[InGame] ThirdPerson 이동 + 필드 NPC 대화 (F 키)

주요 동작

  • 방 세션: NumPublicConnections=4, bIsLANMatch=true, 커스텀 설정 ROOMNAME/HOSTNAME
  • RoomList: RoomId 기준 동기화 — 새 방 추가·기존 갱신·최신 검색에 없는 방 제거
  • OSS 는 Online::GetSubsystem(GetWorld()) 로 World 기반 조회 — PIE 다중창에서 인스턴스별 세션 분리 (전역 IOnlineSubsystem::Get() 은 광고/검색 충돌 + 유령 세션 원인이었음), 검색 진행 중 재요청 가드

UI 에셋 파이프라인 (ULobbyAssetGenerator)

UE 5.7 Python 에서는 WidgetBlueprint.WidgetTree/RootWidget 이 protected 라 순수 Python 으로 위젯 트리 구성이 불가 → 에디터 전용 C++ 도구(WITH_EDITOR)를 만들어 Python 커맨드릿에서 호출한다.

  • GenerateLobbyAssets() — WBP_RoomEntry(Border 루트) / WBP_Room(준비·시작·나가기) / WBP_Lobby(2컬럼: 로비 + 채팅 슬롯)
  • GenerateRoomWiring() — BP_RoomPC(WBP_Room) / BP_RoomGM
  • GenerateNpcChatAssets() — WBP_NpcChat + WBP_Lobby CDO 배선
  • GenerateInGameAssets() — IA/IMC_Interact, WBP_InGameChat, WBP_InteractPrompt, BP_InGamePC, BP_InGameGM
  • 기존 에셋은 위젯 트리 in-place 재구성(UClass 유지 → 외부 참조 보존), 멱등 실행 가능
  • CDO 기본값은 리플렉션(FClassProperty/FObjectProperty)으로 설정
  • 전 화면 위젯은 1920×1080 기준 풀스크린 앵커(0,0→1,1) 레이아웃
  • InGame 맵 복제/NPC 배치/GameMode Override 는 Python 드라이버가 수행

엔드투엔드 테스트 방법

# 1) 계정 서버 기동 (Ubuntu ~/VibeCoding, MySQL game_account DB 켜진 상태)
uvicorn app.main:app --host 0.0.0.0 --port 8088

# 2) LLM 서버 기동 (같은 PC): llama.cpp llama-server + Gemma, 포트 8080 (--reasoning-budget 0 권장)

# 3) 에디터 Play 설정: Number of Players = 2, Net Mode = Play As Listen Server
  1. 각 창에서 로그인 → Lobby (우측 세라와 채팅 가능)
  2. 호스트 창: 방 이름 입력 → 방 만들기 / 클라 창: 새로고침 → 참가
  3. Room: 클라 준비 → 호스트 시작 → 전원 InGame 이동
  4. InGame: WASD 이동 → NPC 접근 → "[F] 대화 시작" → F → 입력창 클릭 후 질문 → 응답(수십 초) → ESC 또는 (포커스 해제 후) F 로 닫기 → 이동 재개

콘솔 명령 검증: ` 키 → Account.Login ..., Npc.Chat 안녕하세요 → Output Log 의 [Account] / [NpcChat]

참고: LLM 주소가 127.0.0.1 이므로 다른 PC 클라이언트는 LLM 접근 불가 (같은 PC 다중 실행은 정상). 필요 시 Npc.SetLlmUrl 로 변경.
